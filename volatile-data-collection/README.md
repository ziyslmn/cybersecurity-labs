# Volatile Data Collection Investigation

## Objective
The objective of this investigation was to collect and analyze volatile system artifacts from a running Linux system. Volatile data is important in forensic investigations because it disappears when a system is shut down.

## Tools Used

Linux Command Line Utilities  
Process and system monitoring commands  
Network analysis tools  

## Investigation Activities

The investigation involved collecting live system information including:

- Running processes
- Network connections
- Routing tables
- Open files
- Memory maps
- Mounted file systems
- Kernel messages

Additional analysis included identifying active services, examining network interfaces, and performing network scanning to understand system activity.

## Findings

The investigation demonstrated how volatile system data can provide valuable insight into system activity, running processes, and network communication. This information is critical for incident response and digital forensic investigations.

## Skills Demonstrated

- Live system forensic analysis
- Volatile artifact collection
- Process and network analysis
- Incident response investigation techniques
