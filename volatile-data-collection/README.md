# Volatile Data Collection Investigation

## Objective
The objective of this investigation was to collect and analyze volatile artifacts from a running system in order to understand active processes, network connections, and system activity.

## Tools Used

- Linux command line utilities
- System monitoring commands
- Network analysis tools

## Investigation Workflow

1. Identify running processes.
2. Examine open files and system resources.
3. Analyze network connections and routing tables.
4. Review mounted file systems.
5. Examine kernel messages.
6. Collect system activity artifacts.
7. Document findings relevant to system investigation.

## Findings

Volatile system artifacts provided insight into real-time system behavior, including active processes, network activity, and system configuration. These artifacts are essential for incident response investigations.

## Skills Demonstrated

- Live system forensic analysis
- Volatile artifact collection
- Process analysis
- Network activity investigation
